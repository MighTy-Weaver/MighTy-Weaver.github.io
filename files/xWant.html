Commonsense Knowledge Tuple Evaluation - xWant

Hi there! Welcome and thank you for participating this survey!

In this survey, your task is to decide how each xWant Commonsense Knowledge Tuple is likely to happen. Each Commonsense Knowledge is presented in the form of (head, xWant, tail) tuple.

The definition of (head, xWant, tail) is given by: [head], as a result, PersonX want [tail].

Below is a positive example:

(PersonX feels tired, xWant, get some energy)

This is always happening because if PersonX feels tired, then he must want some energy back to feel better. Thus this is a always happening event. On the contrast,

Below is a negative example:

(PersonX go to play the match, xWant, drinks too much coffee)

This is invalid because PersonX go to play the match, so it has no relation with he wants to drink too much coffee, thus it's an invalid knowledge.


For every question below, you should grade the knowledge provided in each question with the instruction above, and the hints below:

1. The knowledge is machine-generated, thus please ignore grammar mistakes, focus ONLY on the main events in both head and tail.

2. If the tuple is meaningful in the way that it can happen in any circumstance, then it is plausible. Otherwise it's not.

3. The Human Readable Text in each question is for reference only, it might not be suitable for every knowledge tuple, please always stick to the tuple.

4. The definition of choices are:

  Full defenition of All choices
 always / often   The knowledge assertion presented is always or often true. 
 sometimes / likely   The knowledge assertion presented is sometimes or likely true. 
 farfetched / never   The knowledge assertion presented is false or far-fetched at best. 
 invalid   The knowledge assertion presented is invalid or makes no sense.
